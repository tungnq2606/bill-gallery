## Project Overview

Bill Gallery — a personal mobile app for saving paper receipts and bank transfer screenshots as a visual gallery. Users scan/import bill images, detect amounts via OCR, split expenses, and track who still owes money. Local-first (SQLite), no account required. Organized by date or trip.

**Tech Stack**: React Native 0.85.3 · Expo SDK 56 · TypeScript strict · Expo Router · Zustand · expo-sqlite · expo-camera · react-native-mlkit-ocr

## Code Style

### Language & Types
- TypeScript strict mode — never use `any`, define explicit types
- Path aliases: `@/` maps to `./src/`, `@/assets/` maps to `./assets/`
- Amounts stored as integers (VND has no decimals: 147000 not 147.000)
- All IDs are UUIDv4 strings (client-generated, sync-friendly)

### Components / Modules
- Functional components with arrow functions
- Shared components in `src/shared/components/` — reusable design system
- Feature components in `src/features/<name>/` — screen-specific
- Custom hooks prefixed with `use` (useOCR, useSplitCalculator)
- Repository pattern: screens → stores → repositories → SQLite
- Zustand for UI state only, SQLite is source of truth

### Naming Conventions
- Files: `PascalCase.tsx` for components, `camelCase.ts` for utils/hooks/stores
- Components: PascalCase (`BillTile`, `PayerSelector`)
- Hooks: camelCase with `use` prefix (`useOCR`, `useSplitCalculator`)
- Stores: camelCase with `Store` suffix (`billStore`, `tripStore`)
- Repositories: camelCase with `Repo` suffix (`billRepo`, `splitRepo`)
- Types/interfaces: PascalCase (`Bill`, `SplitShare`, `Person`)
- Constants: UPPER_SNAKE_CASE for true constants, camelCase for config objects
